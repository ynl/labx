# 🚀 快速启动指南

## 验证应用可用性

### 方法 1: 运行验证脚本（推荐）

```bash
node verify-app.js
```

这将检查：
- ✅ 所有必需文件是否存在
- ✅ 依赖是否正确安装
- ✅ 代码结构是否完整
- ✅ 功能是否实现

### 方法 2: TypeScript 编译检查

```bash
npx tsc --noEmit
```

如果没有输出任何错误，说明代码质量良好。

### 方法 3: 启动开发服务器

```bash
# 1. 创建环境变量文件（如果还没有）
cp .env.example .env

# 2. 启动 Expo
npm start

# 3. 选择平台
# - 按 'w' 在浏览器中打开
# - 按 'i' 在 iOS 模拟器中打开（仅 macOS）
# - 按 'a' 在 Android 模拟器中打开
```

即使不配置 Supabase，应用的 UI 仍然可以正常显示：

## 完整使用流程

### 第 1 步：准备 Supabase（5 分钟）

1. 访问 https://supabase.com
2. 点击 "Start your project"
3. 创建新项目
4. 进入 SQL Editor
5. 复制粘贴 `supabase-schema.sql` 的内容
6. 点击 Run 执行

### 第 2 步：配置环境（1 分钟）

1. 在 Supabase Dashboard → Settings → API
2. 复制 "Project URL" 和 "anon public" key
3. 编辑 `.env` 文件：

```bash
EXPO_PUBLIC_SUPABASE_URL=https://你的项目.supabase.co
EXPO_PUBLIC_SUPABASE_ANON_KEY=你的anon-key
```

### 第 3 步：启动应用（30 秒）

```bash
npm start
```

按 `w` 在浏览器中打开，你将看到：

## 🐛 常见问题

### Q: 看不到数据？
A: 检查 `.env` 配置是否正确，Supabase SQL 是否执行成功。

### Q: 编译错误？
A: 运行 `npm install` 重新安装依赖。

### Q: 图片上传失败？
A: 当前版本使用 URL，实际图片不会上传到 Supabase。要实现真正上传，需配置 Supabase Storage（见 SETUP_GUIDE.md）。

### Q: Web 版样式异常？
A: 确保 `react-native-web` 已安装：`npm install react-native-web react-dom`
